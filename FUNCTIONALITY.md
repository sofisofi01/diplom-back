# Wellness Backend - Описание функционала

## Архитектура системы

### Модули приложения

#### 1. AuthModule & UsersModule
**Функционал:**
- JWT аутентификация с refresh токенами
- Хеширование паролей через bcrypt
- Middleware для защиты маршрутов
- Регистрация и авторизация пользователей

**Модели:**
- `User` - кастомная модель пользователя с email как USERNAME_FIELD

#### 2. ProfileModule
**Функционал:**
- Анкета пользователя: пол, возраст, рост, текущий/целевой вес
- Уровень активности и цели (похудение/поддержание/набор массы)
- Автоматический расчет суточной нормы калорий по формуле Миффлина-Сан Жеора

**Модели:**
- `Profile` - профиль пользователя с расчетными методами

#### 3. FoodDiaryModule & ExternalApiModule
**Функционал:**
- Интеграция с Edamam Nutrition API для поиска продуктов
- Дневник питания с разделением по приемам пищи
- Автоматический подсчет калорий, белков, жиров, углеводов
- Redis кэширование для оптимизации API запросов

**Модели:**
- `FoodItem` - продукты питания
- `FoodDiaryEntry` - записи дневника питания

#### 4. ExercisesModule & WorkoutPlansModule
**Функционал:**
- База упражнений с описаниями, целевыми мышцами, изображениями
- Конструктор персональных планов тренировок
- Структура: План → День тренировки → Упражнения (подходы/повторения/вес)

**Модели:**
- `Exercise` - упражнения
- `WorkoutPlan` - планы тренировок
- `WorkoutDay` - дни тренировок
- `WorkoutExercise` - упражнения в тренировке

#### 5. ProgressModule
**Функционал:**
- Ежедневные замеры веса с заметками
- Отслеживание целей с прогнозированием даты достижения
- Аналитика: графики веса, калорий, соответствие плану
- Линейная регрессия для прогноза прогресса

**Модели:**
- `WeightEntry` - замеры веса
- `GoalProgress` - цели и прогресс

## Технические особенности

### Безопасность
- JWT токены с автоматическим обновлением
- Middleware для проверки аутентификации на защищенных маршрутах
- bcrypt хеширование паролей
- CORS настройки для фронтенда

### Производительность
- Redis кэширование API запросов к внешним сервисам
- Оптимизированные SQL запросы с select_related/prefetch_related
- Пагинация для больших списков данных

### API Design
- RESTful архитектура
- Консистентные HTTP статус коды
- Валидация данных через Django REST Framework serializers
- Фильтрация и поиск по параметрам

### База данных
- PostgreSQL для надежности и производительности
- Миграции Django для версионирования схемы
- Индексы для оптимизации запросов
- Связи между моделями через ForeignKey/ManyToMany

## Интеграции

### Edamam Nutrition API
- Поиск продуктов питания по названию
- Получение данных о питательности
- Кэширование результатов на 1-24 часа

### Redis Cache
- Кэширование поисковых запросов
- Сессии пользователей
- Временное хранение вычислений

## Аналитика и отчеты

### Прогресс пользователя
- Динамика изменения веса
- Соответствие плану питания (калории)
- Прогноз достижения целевого веса
- Средние показатели за период

### Метрики системы
- Использование API внешних сервисов
- Эффективность кэширования
- Активность пользователей

## Масштабируемость
- Модульная архитектура для легкого расширения
- Кэширование для снижения нагрузки на внешние API
- Асинхронные задачи через Celery (при необходимости)
- Готовность к горизонтальному масштабированию
